# 能力：代码块语法高亮

## 新增需求

### 需求：围栏代码块必须显示语法高亮

编辑器必须根据指定的语言标识符对围栏代码块应用语法高亮。

#### 场景：TypeScript 代码块的语法高亮

- **当** 用户编写带有语言标识符 "typescript" 的围栏代码块时
- **则** 关键字、字符串、数字、注释和其他标记必须用适当的颜色高亮显示

#### 场景：JavaScript 代码块的语法高亮

- **当** 用户编写带有语言标识符 "javascript" 或 "js" 的围栏代码块时
- **则** JavaScript 语法必须正确高亮显示

#### 场景：没有语言标识符的代码块

- **当** 用户编写围栏代码块但未指定语言时
- **则** 代码必须以等宽字体显示但不进行语法高亮

### 需求：编辑器必须支持嵌套语言解析

编辑器必须使用 CodeMirror 6 的嵌套解析器系统来解析围栏代码块内的代码。

#### 场景：嵌套解析器初始化

- **当** 编辑器使用 markdown 语言支持初始化时
- **则** markdown 解析器必须配置为启用代码语言支持

#### 场景：从围栏标记检测语言

- **当** 解析器遇到带有语言标识符的围栏代码块时
- **则** 必须加载并应用相应的语言解析器到代码内容

### 需求：编辑器必须支持多种编程语言

编辑器必须支持常见编程语言的语法高亮。

#### 场景：核心语言立即可用

- **当** 编辑器加载时
- **则** JavaScript、TypeScript、CSS、Markdown 和 Shell 语言必须无需额外加载即可使用

#### 场景：扩展语言按需加载

- **当** 用户创建 PHP、Go、Java 或 Python 代码块时
- **则** 必须按需加载相应的语言解析器

#### 场景：识别语言别名

- **当** 用户指定语言为 "js"、"javascript"、"ts" 或 "typescript" 时
- **则** 必须应用正确的语言解析器

### 需求：语法高亮不得影响编辑器性能

语法高亮必须增量应用而不阻塞编辑器。

#### 场景：大型代码块

- **当** 用户有超过 100 行的代码块时
- **则** 语法高亮必须仅应用于可见部分

#### 场景：没有代码块的文档

- **当** 文档不包含围栏代码块时
- **则** 除基础 markdown 解析器外不得加载任何语言解析器

### 需求：语法高亮必须在普通模式和实时预览模式下都能工作

无论编辑器模式如何，代码块都必须显示语法高亮。

#### 场景：普通模式下的语法高亮

- **当** 编辑器处于普通模式 (livePreview=false) 时
- **则** 代码块必须显示语法高亮

#### 场景：实时预览模式下的语法高亮

- **当** 编辑器处于实时预览模式 (livePreview=true) 时
- **则** 代码块必须在保持实时预览样式的同时显示语法高亮
