# 能力：代码块行号

## 架构约束

**关键设计决策**：代码块装饰（包括行号）不依赖 hybridMarkdown 的 active-scope 机制。

- 代码块装饰插件始终开启，独立于 hybridMarkdown
- 光标进入代码块后，行号和背景样式持续保留
- FencedCode 节点已从 hybridMarkdown 的 HYBRID_SCOPE_NODES 中移除
- hybridMarkdown 只负责标题、强调、链接等需要源码态/渲染态切换的结构

## 新增需求

### 需求：代码块必须支持可选的行号

当启用 codeLineNumbers 属性时，代码块必须显示行号。

#### 场景：启用行号

- **当** codeLineNumbers 属性为 true 时
- **则** 代码块中的每一行必须显示行号

#### 场景：默认禁用行号

- **当** codeLineNumbers 属性未设置或为 false 时
- **则** 代码块不得显示行号

### 需求：行号必须从 1 开始

代码块中的行号必须始终从 1 开始，每个代码块独立计数。

#### 场景：第一行行号

- **当** 代码块启用行号时
- **则** 代码的第一行必须编号为 1

#### 场景：多个代码块

- **当** 文档包含多个带行号的代码块时
- **则** 每个代码块必须独立地从 1 开始编号

### 需求：行号必须排除围栏标记

行号必须仅显示实际代码内容，不包括围栏标记。

#### 场景：围栏标记不编号

- **当** 代码块以 ``` 和语言标识符开始时
- **则** 围栏标记行不得获得行号

#### 场景：结束围栏不编号

- **当** 代码块以结束的 ``` 结束时
- **则** 结束围栏标记不得获得行号

### 需求：行号必须具有独特的样式

行号必须在视觉上与代码内容区分开来。

#### 场景：行号外观

- **当** 显示行号时
- **则** 行号必须使用比代码文本更浅的颜色
- **则** 行号必须右对齐
- **则** 行号必须与代码内容有适当的间距

#### 场景：行号不可选择

- **当** 用户选择代码文本时
- **则** 行号不得包含在选择中

### 需求：行号必须通过 CSS 伪元素渲染

行号必须使用 `data-code-line-number` 属性配合 `::before` 伪元素渲染，而非 DOM widget。

#### 场景：行号实现方式

- **当** 启用行号时
- **则** 代码行必须具有 `data-code-line-number` 属性
- **则** 行号必须通过 CSS `::before` 伪元素显示
- **则** 行号不得作为 widget 或其他 DOM 节点插入文本流

#### 场景：行号不影响文本流

- **当** 用户选择代码文本时
- **则** 行号不得包含在选择中
- **当** 用户定位光标时
- **则** 行号不得影响光标列位置计算

### 需求：行号必须适应主题

行号颜色必须尊重当前代码块主题。

#### 场景：深色主题中的行号

- **当** 代码块使用深色主题时
- **则** 行号必须使用适当的浅色以确保可见性

#### 场景：浅色主题中的行号

- **当** 代码块使用浅色主题时
- **则** 行号必须使用适当的深色以确保可见性

### 需求：行号必须与语法高亮配合工作

行号和语法高亮必须协同工作而不产生冲突。

#### 场景：组合行号和高亮

- **当** codeLineNumbers 为 true 且语法高亮处于活动状态时
- **则** 代码必须正确显示行号和语法颜色

### 需求：行号更改必须立即生效

对 codeLineNumbers 属性的更改必须立即生效，无需重新加载编辑器。

#### 场景：动态切换行号

- **当** codeLineNumbers 属性从 false 更改为 true 时
- **则** 行号必须立即出现在所有代码块中

#### 场景：禁用行号

- **当** codeLineNumbers 属性从 true 更改为 false 时
- **则** 行号必须立即从所有代码块中消失
